## Why artifacts matter

Artifacts help prove that each stage executed.

They also help reviewers inspect:

```text
tool output
JSON reports
pipeline results
security findings
evidence files
```

## Evidence review

Before publishing evidence, I reviewed the outputs to avoid exposing sensitive values.

Review checks included:

```text
No real secrets
No private tokens
No customer data
No Docker image tar files
No unnecessary archives
Redacted Gitleaks output
```

## Large report files

Some Trivy JSON reports are large.

This is expected because container image scans can produce many vulnerability records.

The project keeps these reports as evidence, while avoiding Docker image tar files.

## Conclusion

Artifacts and reports are a key part of this project because they turn tool execution into reviewable evidence.
