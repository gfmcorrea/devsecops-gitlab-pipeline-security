# Pipeline Stages

This file documents the stages used in the GitLab CI/CD pipeline.

## 1. setup

Purpose:

```text
Prepare the demo application for scanning.

Main checks:

Node.js version
npm version
npm install
npm test

This stage confirms that the application dependencies can be installed before running security checks.

2. sast

Purpose:

Run static application security testing.

Tool:

Semgrep

Main check:

semgrep scan --config p/javascript app

This stage looks for insecure code patterns in the application source code.

3. secret_scan

Purpose:

Detect hardcoded secrets or sensitive patterns.

Tool:

Gitleaks

Main check:

gitleaks detect --source . --config .gitleaks.toml --redact --no-git

This stage uses a fake demo secret for safe validation.

4. dependency_scan

Purpose:

Identify vulnerable application dependencies.

Tool:

npm audit

Main check:

npm --prefix app audit

This stage checks Node.js dependencies for known vulnerabilities.

5. container_build

Purpose:

Build the Docker image for the demo application.

Tool:

Docker

Main check:

docker build -t devsecops-demo-app:$CI_COMMIT_SHORT_SHA app

This stage creates the container image that will be scanned by Trivy.

6. container_scan

Purpose:

Scan the Docker image for known vulnerabilities.

Tool:

Trivy

Main check:

trivy image --input devsecops-demo-app-ci.tar

This stage reviews the image layers, operating system packages, and application components.

7. report_summary

Purpose:

Summarize generated evidence files.

This stage lists artifacts generated by the pipeline and reminds me to review all scan outputs before confirming findings.
